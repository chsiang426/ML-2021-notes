將訓練資料輸入進 FFF﻿，FFF﻿ 直接輸出一個模型 f∗f^*f∗﻿ 可以直接進行測試

2. 尋找 Learning Algorithm 三步驟

注意：

任務有訓練任務與測試任務之別

Step 1：What is learnable？

決定 learning algorithm 中要被學的 components（網路架構、初始參數、學習率等等），以 ϕ\phiϕ﻿ 表示

不同的 meta learning 方法的差異在於 components 的選擇

Step 2：Define loss function L(ϕ)L(\phi)L(ϕ)﻿

訓練資料來自很多訓練任務，每個任務中有訓練集和測試集

定義 loss function L(ϕ)L(\phi)L(ϕ)﻿：

  1. 將某一任務的訓練資料輸入進 learning algorithm FϕF_\phiFϕ​﻿，得到模型 fθ1∗f_{\theta^{1*}}fθ1∗​﻿
  1. 使用對應任務的測試資料對模型 fθ1∗f_{\theta^{1*}}fθ1∗​﻿ 進行測試，計算每個預測資料的結果與 ground truth 之間的 cross entropy，並將全部的 cross entropy 加總得到 l1l^1l1﻿
    • l1l^1l1﻿ 若越小，表示模型 fθ1∗f_{\theta^{1*}}fθ1∗​﻿ 越好，代表是好 learning algorithm FϕF_\phiFϕ​﻿
    • l1l^1l1﻿ 若越大，表示模型 fθ1∗f_{\theta^{1*}}fθ1∗​﻿ 越不好，代表是差 learning algorithm FϕF_\phiFϕ​﻿
  1. 將下一任務的訓練資料輸入進 learning algorithm FϕF_\phiFϕ​﻿，得到模型 fθ2∗f_{\theta^{2*}}fθ2∗​﻿，並計算每個預測資料的結果與 ground truth 之間的 cross entropy，並將全部的 cross entropy 加總得到 l2l^2l2﻿
  1. 以此類推得到全部訓練任務的 lll﻿，並加總得到 learning algorithm 的 loss L(ϕ)L(\phi)L(ϕ)﻿

注意：

在一般機器學習中，loss 是根據訓練資料得來的；而在 meta learning 中，loss 是根據訓練任務中的測試資料得來的

Step 3：Optimazation

  • 若 ∂L(ϕ)∂ϕ\frac{\partial L(\phi)}{\partial \phi}∂ϕ∂L(ϕ)​﻿ 可微，則可以使用 gradient descent 找出 ϕ∗\phi^*ϕ∗﻿ 最小化 L(ϕ)L(\phi)L(ϕ)﻿
  • 若 ∂L(ϕ)∂ϕ\frac{\partial L(\phi)}{\partial \phi}∂ϕ∂L(ϕ)​﻿ 不可微（ϕ\phiϕ﻿ 有可能是一個 network 架構），使用 RL 硬 train，或其他方法

最終得到一 learning algorithm Fϕ∗F_{\phi^*}Fϕ∗​﻿ 使 L(ϕ)L(\phi)L(ϕ)﻿ 最小化

我們真正關心的是在測試任務上，learning algorithm Fϕ∗F_{\phi^*}Fϕ∗​﻿ 的性能

將測試任務中的訓練資料輸入進 learning algorithm Fϕ∗F_{\phi^*}Fϕ∗​﻿ 進行訓練得到模型 fθ∗f_{\theta^*}fθ∗​﻿，fθ∗f_{\theta^*}fθ∗​﻿ 就是我們最終想要的模型

3. ML vs Meta

3.1 Goal

  • ML：找到一個能完成任務的函數 fff﻿
  • Meta：找到一個 learning algorithm FFF﻿，能夠找到能完成任務的函數 fff﻿

3.2 Training Data

  • ML：使用一個任務中的訓練資料進行訓練
  • Meta：使用若干個訓練任務進行訓練，每個訓練任務中都有訓練資料及測試資料
    • Support set：訓練任務中的訓練資料
    • Query set：訓練任務中的測試資料

3.3 Framework

3.3.1 Training

  • ML：人為設定 learning algorithm，稱作 Within-task Training
  • Meta：多個任務上訓練得到 learning algorithm，稱作 Across-task Training

3.3.2 Testing

  • ML：直接使用訓練得到的模型在任務中對測試資料進行測試，稱作 Within-task Testing
  • Meta：需要測試的是 learning algorithm，稱作 Across-task Testing
    • learning algorithm 以測試任務的訓練資料做為訓練，稱作 Within-task Training
    • 以測試任務的測試資料測試模型，稱作 Within-task Testing

    Episode = Within-task Training + Within-task Testing

3.3.3 Loss

  • ML：對一個任務中所有的測試數據的損失之和
  • Meta：lll﻿ 是一個訓練任務的 loss，LLL﻿ 是所有訓練任務的 loss 總和

計算一個 lll﻿，需要一次的 Within-task Training + Within-task Testing 即一個 episode。將 Within-task Training 稱作 Inner Loop；Across-task training 稱作 Outer Loop

4.1.1 MAML vs Pre-training

  • MAML 需要用到有標註的資料
  • pre-training（self-supervised learning）使用的資料沒有標註

注意：

meta learning 中所謂不同任務的訓練，實際上就是不同的 domain，所以也可以說 meta learning 是 domain adaptation 的一種方法

過去 pre-training 還有其他的方法，如將來自不同任務的資料混在一起進行訓練（稱作 multi-task training）

multi-task training 通常作為 meta-learning 的 baseline

4.3.1 解法 1：Reinforcement Learning

  • ϕ\phiϕ﻿：the agent's parameters

  • Reward to be maximized：−L(ϕ)-L(\phi)−L(ϕ)﻿
